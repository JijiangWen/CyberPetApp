# 第 17 章：喂食器、饮水泉与猫咪状态自然衰减 🍲💧

猫咪作为游戏世界里最核心的陪伴实体，拥有自身的饱食度、水分、精力、快乐度与健康度等状态指标。在玩家离线或专注捕鱼期间，这些状态会随时间缓慢流逝。为了避免玩家每次上线猫咪都奄奄一息，系统引入了**自动喂食器**、**自动饮水泉**以及**状态自然衰减-自动治愈**系统。

---

### 17.1 猫咪五维指标的自然衰减与健康度公式

游戏心跳（每 2 秒一次的 Tick）中，系统会调用猫咪实体的 `Tick` 逻辑进行状态的逐级扣减：

#### 1. 基础衰减速度：
* 饱食度（Hunger）：基础每 Tick 衰减 1 点。
* 水分（Thirst）：基础每 Tick 衰减 2 点（猫咪更容易口渴）。
* 精力（Energy）：活动状态下扣除对应能耗；非活动挂机状态下，若处于“客厅”且有沙发被动，会以微弱速率自然恢复。
* 快乐度（Happiness）：如果饱食度或水分充足，被动自然恢复；若饥饿度低于 300，快乐度开始以每 Tick 3 点的速度暴跌。

#### 2. 健康度（Health）恶化与致死公式：
健康度是衡量猫咪生存安全的核心指标（满值 1000）。健康度不会平白无故下降，只有在以下两种极端情况下才会恶化：
* **极度饥饿**：饱食度归 0。
* **极度脱水**：口渴值归 0。

如果这两项指标有任一归零，健康度将以每 Tick 15 点的超高速率发生**直接坏死下降**。当健康度跌破 500 点时，猫咪会进入“生病”动画状态，打工收益直接降为 0；如果健康度跌至 0，猫咪将陷入濒死昏迷，必须花费 30g 购买“宠物医用急救包（CatTreat）”方可重获新生。

### 17.2 智能自动喂食器（Auto Feeder）与自动补水泉（Auto Waterer）

为了给挂机玩家减负，当玩家解锁了厨房对应的“智能喂食器”和“宠物饮水泉”家具后，系统在心跳循环中会开启**托管托管机制**。

#### 1. 自动填粮（AddFoodFromBackpackAsync）：
玩家可以从背包中把食物（如“普通猫粮”或不带持续状态 Buff 的玩家自烹饪食物）装填进自动喂食器中。
* **装粮加工费（FeederProcessingFee）**：每装入 1 份食物需要扣除 2g 金币加工费。
* **Slot 槽重排（SyncSlotsAfterFeedAsync）**：当自动喂食触发时，为保证喂食器内的食物卡片顺序不发生错乱，系统在数据库表 `FeederFoods` 中会动态重排 `SlotIndex`，并按“先进先吃”的先进先出（FIFO）机制扣减最前排口粮。
* **一键批量装填（BatchAddFood）**：允许玩家点击一次，自动筛选背包中所有合法的食品、零食，根据格子上限一键全部填满，极大提升了中后期的游戏便利性。

#### 2. 自动喂食/补水触发机制：
在心跳 Tick 执行时：
```csharp
// 自动喂食检查
if (cat.Hunger < 600 && feeder.FoodCount > 0)
{
    // 自动取出最前排的口粮
    var food = feeder.ConsumeFood(); 
    // 跨服务执行：猫咪吃食并恢复饱食度、精力和快乐
    cat.Eat(food); 
    // 异步更新数据库 Feeder 表并重排槽位
    _ = _feederService.SyncSlotsAfterFeedAsync(feeder);
}

// 自动饮水检查
if (cat.Thirst < 600 && waterer.WaterCount > 0)
{
    var water = waterer.ConsumeWater();
    cat.Drink(water);
    _ = _watererService.SyncSlotsAfterDrinkAsync(waterer);
}
```

### 17.3 离线托管补偿的喂食计算

值得注意的是，在**离线补偿**章节（第 6 章）中，自动照顾系统同样生效。

当系统在补偿循环中模拟 900 次 Ticks 时，每一轮循环都会实时提取玩家的自动喂食器和饮水泉存量。如果发现猫咪属性跌破临界点，离线逻辑会**自动模拟喂食、扣减喂食器数据库数量、重新累加猫咪属性**。这保证了即使用户下线 10 小时，只要下线前喂食器里的干粮装填充足，上线后猫咪依然保持着高饱食和高健康度的绝佳状态！
